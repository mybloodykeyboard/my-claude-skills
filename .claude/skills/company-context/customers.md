## 시장 포지셔닝

- **B2G + B2B 중심**. ESG(환경·사회·지배구조) 관점 사업 전개.
- B2C·B2G2C 사례 일부 (마음닥, WiseRun 등).
- 글로벌 확장 시작 — 페루정부 협력과제 (2025).

## 주력 도메인 Top 7 + 페르소나·페인포인트

### 1) 건설

- **대표 고객**: 포스코이앤씨.
- **페인포인트**:
  - 현장 데이터·암묵지가 비구조화·비정형 → 검색·재활용 어려움.
  - 공정·품질·원가 관리에 산재된 시스템·데이터 — Agent 기반 통합 필요.
  - 전문용어(예: PHC파일) 범용 LLM이 인식 못함.
  - 하자 발생 시 원인·조치 사례 검색이 사람 의존.
- **자주 만나는 RFP 패턴**: 현장 특화 AI 어시스턴트, 하자예방, 품질 RAG 챗봇.
- **랩큐의 무기**: 우리현장 GPT (Multi-Agent), 자체 임베딩 + 건설지식 코퍼스, 하자예방기술 추천.

### 2) 석유화학 / 제약 (SK캐미칼군)

- **대표 고객**: SK캐미칼 (석유화학·제약 부문).
- **페인포인트**:
  - 설비·공정·안전(SHE) 지식이 도면·매뉴얼 PDF에 흩어짐 → 검색·인용 어려움.
  - 영업 활동 기록·이력이 개인 단위 — 팀 간 공유·Compliance 위배 감지 부재.
  - 생산공정 문제 해결에 데이터 분석 역량 부족.
- **자주 만나는 RFP 패턴**: 설비포털 어시스턴트, 영업지원 LLM 플랫폼, 생산공정 분석 과제.
- **랩큐의 무기**: 멀티모달 RAG (도면), Vertical RAG, Agent 기반 영업 자동화.

### 3) 자동차 산업

- **대표 고객**: 한국자동차연구원.
- **페인포인트**:
  - 산업 저널·논문·기업정보 분산 → Research 작업이 수동.
  - 보안성 유지 필요 → 외부 LLM에 사내 데이터 노출 불가.
- **자주 만나는 RFP 패턴**: Research Agent, 문서 자동 생성 (PROP), 보고서 작성 어시스턴트.
- **랩큐의 무기**: Customized PROP, 자동차산업 벡터 DB, 사내 데이터 보안성 유지 (자체 VDB).

### 4) 공공 / 행정 / 민원 (가장 두꺼운 도메인)

- **대표 고객**: 고양시, 화성시, 한국항공안전기술원, 페루정부, 병무청, 과기정통부, 문화부, 국립국어원, 4차산업혁명위.
- **페인포인트**:
  - 민원·행정 절차가 복잡 — 시민이 접근 어려움.
  - 데이터 기반 행정 전환 필요 (다년간 컨설팅 수요).
  - 공공 LLM 도입 시 신뢰성·근거·설명가능성 강하게 요구됨.
  - 다국어 (한-페루 등).
  - 다채널 민원 분산 (전화·챗봇·게시판), 응대 품질 불균형, 9-to-6 한계, 데이터 사일로.
  - 동시 접속 부하 시 응답 속도 저하 (RPU/FPGA 가속의 시장 근거).
  - 외국인 민원 급증 — 2025년 국내 체류 외국인 250만명+ (전체 인구의 5%+), 최근 3년간 외국인 민원 접수 60% 증가. 지정 창구·다언어 인력 확보의 한계, 소통 오류 리스크. (A-tran 협업의 시장 근거)
- **자주 만나는 RFP 패턴**: ISP/AX 로드맵 수립, 민원 챗봇, 행정 검토 Agent, 데이터 공동활용 방안 컨설팅, 24x365 보이스 AI 상담, 지자체장 대시보드(데이터 기반 행정).
- **랩큐의 무기**: 온톨로지 기반 설명가능성, 다국어 LoRA fine-tuning, 컨설팅+PoC 결합 경험, 고양시 검증된 정량 성과(처리량 280%↑, 만족도 85→97%), RPU/FPGA 가속(동시 접속 강함).

#### 4-1) 지자체장 페르소나 (AI 정책 보좌관 타겟)

- **핵심 고민**: "하루에도 수백 건, 그 모든 현안을 파악하고 있는가?" — 민원 폭주 대응 지연, 데이터 취합 수일, 일방향 행정 홍보로 민심 괴리.
- **보는 포인트**:
  - 출근 30초 브리핑·집무실 상세 열람 듀얼 인터페이스.
  - 데이터 기반 객관적 정책 근거 (경험·직관 의존 탈피).
  - 양방향 소통 + 만족도 즉시 확인.
  - 24x365 응대 시간.
- **지역별 시그니처 시나리오** (보유):
  - **노원구**: 재개발·재건축 갈등, 청년 소통(광운대·과기대), 환경(공릉천·불암산).
  - **중랑구**: 중랑천 환경, 노후 주거 정비, 복지 사각지대.
  - **대전 서구**: 도안신도시 vs 원도심 균형, 둔산 학원가 주차, 신규 전입.
  - **대전 유성구**: 외국인 연구자 다국어 응대, 유성온천 관광 예측, 첨단 기술 친화 UX.
  - **완주군**: 스마트 농장 비서, 귀농귀촌 정착 케어, 어르신 음성 민원.
  - **용인시**: 반도체 클러스터, 수지·기흥 광역버스, 처인구 도로 인프라.

### 5) 환경 / 인프라 (ESG 환경)

- **대표 고객**: 그린텍 아이엔씨, 한국융합아이티 (수처리 영역).
- **페인포인트**:
  - 하수 수위·수량 예측이 강우·기후 변화에 취약.
  - 비점오염원 식별 인력 부족.
  - 탄소배출량 예측 — LCA 데이터 활용 미흡.
- **자주 만나는 RFP 패턴**: 환경 인프라 예측 모형, 객체 탐지 이미지 AI, LCA 분석.
- **랩큐의 무기**: 하수 관련 특허 2건, 비점오염원 이미지 데이터셋 + 모델 보유.

### 6) 의료 / 돌봄 / 정신건강

- **대표 고객**: 화성시, 현소소망의료재단, DrForest, 시니어 아파트.
- **페인포인트**:
  - 정신건강 위험군 사전 탐지 어려움 (자가 보고 의존).
  - 시니어·청소년·1인 가구의 사회적 고립 — 비대면 모니터링 수요.
  - 방문 요양 보호사·요청자 매칭 비효율.
- **자주 만나는 RFP 패턴**: 비대면 정신건강 모니터링, 청소년·시니어 챗봇, 매칭 알고리즘.
- **랩큐의 무기**: 마음닥 (특허), 디지털 바이오마커, B2G2C 운영 경험.

### 7) 보안 / 정보보호

- **대표 고객**: 대법원, 한국재정정보원, KCA, 교보증권, NIPA.
- **페인포인트**:
  - 내부자 보안위협 — 사용자 행동패턴 분석 필요.
  - 보안관제 시스템 고도화 — AI 적용 사례 부족.
  - 금융권 마이데이터 — 정보보호 가이드·시큐어코딩.
- **자주 만나는 RFP 패턴**: 보안관제 컨설팅, ISMS-P 기술지원, 모의해킹·취약점 점검.
- **랩큐의 무기**: ISMS-P 경험 (대법원 전자소송), AI 기반 이상행동 탐지, NIPA AI 보안관제 방법론.

## 자주 등장하는 RFP 평가항목 패턴 (도메인 무관)

랩큐 솔루션이 자주 응답해야 하는 RFP 평가 축:

1. **도메인 이해·전문성**: 발주처 도메인 사례, 도메인 코퍼스 보유 여부.
2. **LLM 신뢰성**: 할루시네이션 방지, 근거 기반 응답, 자체 검증 지표.
3. **데이터 보안**: 자체 VDB·온프레미스 옵션, LoRA fine-tuning으로 데이터 외부 노출 최소화.
4. **실행 능력**: 유사 도메인 구축 사례 수, 운영 안정성.
5. **확장성**: Multi-Agent, function calling, 이종 DB 통합.
6. **설명가능성 (XAI)**: 공공·금융 RFP에서 강하게 요구됨.

## 의사결정자 페르소나

| 페르소나 | 보는 포인트 |
|---|---|
| 공공 발주 담당관 / 사업관리자 | 평가항목 충족, 산출물 정의 명확성, 일정 준수, 유사 공공 경험 |
| 현업 부서장 (건설·화학·자동차 등) | 실제 현장 문제 해결력, 도메인 용어 인식, 운영 후 변화 |
| CIO / 정보화 책임자 | 보안, 통합 아키텍처, 기존 시스템 연계, 확장성 |
| 연구원·기술자 (자동차연구원 등) | 모델 정확도, 데이터 처리 방식, fine-tuning 옵션 |
| ESG·복지 담당 (지자체·재단) | 사회적 임팩트, 시범 운영 결과, 사후 운영 부담 |

## 고객이 자주 쓰는 용어 (제안서에 인용하면 공감 신호)

- **건설**: PHC파일, 공종, 자재, 시점, 하자유형, 공정관리, 품질관리, 원가관리, 백호.
- **석유화학**: CDP-4, 캔디드 보일러, SHE(Safety/Health/Environment), 공정·설비 기술문서.
- **자동차**: 산업 저널, 연구보고서, 보안성, 사내 데이터.
- **공공**: 민원, 행정 검토, 디지털 정부, AX, ISP, 정보화전략계획.
- **환경**: 비점오염원, 하수 수위·수량, LCA(전과정평가), 탄소배출.
- **의료**: 디지털 바이오마커, 위험군 발굴, 비대면 케어, 사회적 고립.

> 발주처가 RFP에 쓴 용어를 그대로 인용하는 것이 가장 강한 공감 신호. 제안서 작성 시 RFP 원문에서 도메인 용어 먼저 추출 후 본문에 자연스럽게 녹일 것.
