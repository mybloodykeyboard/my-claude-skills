# Track Record

도메인별 구축 사례 — 제안서에서 가장 강력한 무기.

> _아직 채워지지 않음._

## Suggested structure (per case)

- 도메인 (예: 금융, 제조, 공공, 의료)
- 고객사 (공개 가능한 경우 명시, 아니면 "○○ 대형 보험사" 식으로)
- 기간
- 우리가 한 일 (한 줄)
- 사용 기술·제품
- 성과 (수치 + 측정 조건)
- 후속 (확장 계약, 레퍼런스 가능 여부)

## Tips

- 같은 도메인 사례가 3건 이상이면 "○○ 도메인 다수 구축 경험" 헤드라인 가능.
- 수치는 출처·기준선 명시 ("정확도 95%" → "기존 대비 X% → Y% 개선, 측정조건: …").
